In the first iteration, testing focused primarily on unit testing and verifying that individual components worked correctly. The goal was to identify defects early while the system was still small and manageable. As additional features were added in later iterations, testing expanded to include integration testing to verify that components interacted correctly with one another.
By the final iteration, the testing focus shifted toward regression testing and end-to-end workflow validation. Since multiple features were now connected, changes in one area could affect other parts of the system. Regression testing helped ensure that existing functionality continued to work after updates were made. This progression demonstrates how testing evolves alongside development and supports continuous delivery by identifying issues early, maintaining software quality, and reducing the risk of defects reaching production.
